# 🍽️ RestaurantCRM

Десктопна система управління рестораном із розмежуванням ролей — клієнт, працівник та адміністратор — побудована на Qt6 та PostgreSQL.

---

## 📐 Архітектура проєкту

```
RestaurantCRM/
├── src/
│   ├── backend/       # Серверна логіка взаємодії з БД (по ролях)
│   ├── core/          # Підключення до БД, спільні утиліти
│   └── ui/            # Qt-вікна та форми (.cpp / .h / .ui)
└── CMakeLists.txt     # Конфігурація збірки CMake
```

---

## ✨ Функціональність

### Для клієнтів
- Реєстрація та авторизація (логін + пароль)
- Перегляд меню з фільтрацією за категоріями
- Додавання страв до кошика та оформлення замовлення
- Перегляд поточного кошика з кількістю та сумою

### Для працівників
- Перегляд усіх замовлень на поточний день
- Фільтрація замовлень за часовим діапазоном
- Пошук замовлень конкретного клієнта
- Зміна статусу замовлення (наприклад: прийнято → виконано)

### Для адміністраторів
- Управління користувачами: перегляд, редагування полів, видалення, пошук
- Управління меню: додавання/редагування страв (назва, опис, ціна, вага, категорія, доступність)
- Управління замовленнями: фільтрація за датою та клієнтом, зміна статусу, видалення
- Створення облікових записів для адміністраторів та працівників
- Формування звітів

## 🗄️ Ініціалізація бази даних

У корені проєкту знаходиться файл `init.sql`. Він містить готовий SQL-скрипт для автоматичного створення всіх необхідних таблиць (`Users`, `Menu`, `Orders`) та заповнення їх тестовими даними (категорії меню, тестові облікові записи клієнтів, працівників та адміністраторів).

---

## ⚙️ Налаштування підключення до бази даних

Підключення до БД реалізоване через клас-singleton `dbConnection`. За замовчуванням застосунок використовує ODBC DSN з іменем `RestaurantDSN`.

Для налаштування прямого підключення до PostgreSQL відредагуйте файл `src/core/dbconnection.cpp` (за зразком `dbconnection_example.cpp`):

```cpp
sqlServer = QSqlDatabase::addDatabase("QPSQL");
sqlServer.setHostName("your_host_pooler");
sqlServer.setPort(5432);
sqlServer.setDatabaseName("your_db_name");
sqlServer.setUserName("your_user");
sqlServer.setPassword("your_password");
sqlServer.setConnectOptions("sslmode=require");
```

> Файл `dbconnection_example.cpp` містить готовий приклад — скопіюйте його вміст до `dbconnection.cpp` та заповніть свої дані.

## 📁 Структура проєкту

```
src/
├── main.cpp                          # Точка входу, ініціалізація БД та QApplication
│
├── core/
│   ├── dbconnection.h / .cpp         # Singleton-підключення до PostgreSQL
│   ├── dbconnection_example.cpp      # Приклад конфігурації підключення
│   └── generic_utils.h               # Спільні структури (orderInfo)
│
├── backend/
│   ├── server_login_creation.h/.cpp  # Авторизація, реєстрація, визначення ролі
│   ├── server_user.h/.cpp            # Логіка клієнта: меню, оформлення замовлення
│   ├── server_employee.h/.cpp        # Логіка працівника: замовлення, статуси
│   └── server_admin.h/.cpp           # Логіка адміна: CRUD користувачів/меню/замовлень
│
└── ui/
    ├── mainwindow.h/.cpp/.ui         # Вікно авторизації (вхідна точка UI)
    ├── creationacc.h/.cpp/.ui        # Форма реєстрації нового клієнта
    ├── userwindow.h/.cpp/.ui         # Головне вікно клієнта
    ├── menuwindow.h/.cpp/.ui         # Меню з категоріями та кошиком
    ├── myorders.h/.cpp/.ui           # Перегляд та підтвердження замовлення
    ├── employeewindow.h/.cpp/.ui     # Вікно працівника
    └── adminwindow.h/.cpp/.ui        # Адмін-панель
```

## 🗂 Категорії меню

Система підтримує розгалужену структуру меню з такими категоріями:

**Їжа:** салати, перші страви, основні страви, гарніри, піца, бургери, закуски, десерти

**Суші:** футомакі, каліфорнія, філадельфія, оригінал, дракон, сети

**Напої (безалкогольні):** кава, чай, молочні коктейлі, фреші, лимонади, смузі, холодні напої

**Напої (алкогольні):** пиво, вино, коктейлі, бренді, горілка, джин, текіла, ром, віскі

## 👤 Ролі користувачів

- **Клієнт (`user`)** — реєстрація, перегляд меню, оформлення замовлень
- **Працівник (`employee`)** — перегляд та обробка замовлень на поточний день
- **Адміністратор (`admin`)** — повний доступ: управління користувачами, меню, замовленнями та звітами
